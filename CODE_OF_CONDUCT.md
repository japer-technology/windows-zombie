# Code of Conduct

Ubuntu Zombie is a project about powerful local administration tools. That
work requires clear, respectful collaboration and careful handling of safety
concerns.

## Expected behaviour

- Be respectful and constructive.
- Assume good intent, but accept corrections quickly.
- Keep discussion focused on the project and its users.
- Be careful with security reports, credentials, logs, and screenshots.
- Welcome newcomers and explain trade-offs without gatekeeping.

## Unacceptable behaviour

- Harassment, threats, insults, or discriminatory language.
- Publishing another person's private information.
- Sharing secrets, exploit details, or vulnerable configurations in public
  issues when responsible disclosure is appropriate.
- Repeatedly derailing technical discussions after maintainers ask you to
  stop.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
or block contributors whose behaviour harms the project or community.

Report security issues through `SECURITY.md`. Report conduct concerns to the
repository maintainers by private channel when possible.
